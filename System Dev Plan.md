# 脑域链医开发流程任务清单（完整实现版）

## 状态说明
- `已完成`：仓库中已有代码、脚本、接口或可重复验证结果。
- `进行中`：已有骨架、样板链路或本地联调能力，但未达最终目标。
- `未开始`：仍停留在规划层，仓库中尚未实现。

## 摘要
- 目标是从空仓库落地一套可运行、可测试、可演示的全栈系统，完整覆盖 EEG 安全上传、链上存证、DID/RBAC 授权、联邦学习、审计追溯、数据销毁。
- 交付形态固定为一个 monorepo，包含合约层、后端编排层、EEG/联邦学习 Python 服务、Vue 前端、基础设施编排、技术与安全文档。
- 节奏按 8 周推进，前 6 周完成功能闭环，后 2 周完成安全加固、验收材料和演示准备。

## 当前落地情况（截至 2026-04-20）
- `已完成` 工程骨架、前后端样板链路、Python 服务骨架、合约骨架、共享 OpenAPI、环境文档已经落地。
- `已完成` 上传链路已支持 `EDF/GDF/BDF` 文件校验、链下存储、链上存证摘要生成、详情页查看。
- `已完成` 链访问已抽象为 `ChainGateway`，本地已跑通 `FISCO BCOS 3.x Java SDK` 真实写链。
- `已完成` 存储访问已抽象为 `StorageGateway`，当前支持 `local` 与 `minio` 两种实现。
- `进行中` 前端详情页、脑区活跃度播放、链上状态展示已可演示，但 3D 真脑模型、认证授权、联邦学习、审计中心仍未完成。
- `未开始` DID/VC、RBAC 零信任、FATE 联邦训练、销毁闭环、监管查询、安全加固与最终验收材料。

## 对外接口与关键类型
- `进行中` 后端当前已真实提供 `/datasets` 与 `/system`，并通过后端代理 `brain-activity`；`/auth`、`/access-requests`、`/trainings`、`/audits`、`/destructions` 尚未实现。
- 合约事件固定为：`DataRegistered`、`AccessGranted`、`AccessRevoked`、`TrainingRecorded`、`ContributionRecorded`、`DataDestroyed`、`AuditEmitted`。
- `进行中` 关键业务对象中，`Dataset` 与 `DataAssetProof` 已形成样板实现，其余对象仍在规划。
- `进行中` Python 服务当前已分别承担 `brain-activity` 样板输出与联邦服务占位职责；Java 服务已承担上传编排、链交互、存储编排和状态汇总。
- `进行中` 前端仍采用单一站点，但目前尚未按四类角色切换完整视图。

## 测试与验收
- 单元测试必须覆盖合约方法、哈希/加密工具、权限判断、上传校验、训练任务状态机、审计聚合逻辑。
- 集成测试必须覆盖五条主链路：安全上传并存证、授权申请并审批、带授权的数据发起联邦训练、监管审计追溯、授权撤销与数据销毁。
- 安全测试必须验证：无效签名被拒绝、篡改文件无法通过哈希校验、过期凭证不可访问、越权角色无法查询敏感资产、重放请求不会重复上链。
- 性能验收默认目标：109 名受试者数据可批量导入；存证成功率 100%；溯源查询成功率 100%；核心分类准确率不低于设计文档中的 85% 目标。
- 发布验收标准固定为：本地一键启动成功、完整演示脚本跑通、核心报告齐全、关键安全缺陷清零。

## 假设与默认
- 当前仓库从零开始实施，不复用现成业务代码或现成前后端脚手架。
- 首版即按 `Design.md` 的完整能力实现，不降级为 MVP，不把 DID、联邦学习、零信任或销毁能力延后到二期。
- 开发环境默认使用容器化编排；生产化高可用、跨机房部署和真实医院系统对接不纳入本轮任务，但接口和部署文档需预留扩展说明。
- DID 采用平台内自建注册与 VC 签发方案，零信任落在“统一网关鉴权 + 凭证校验 + 链上角色校验 + 全量审计”这一组合上。
- 所有原始 EEG 数据仅以加密形式进入存储层，链上只存哈希、指纹、权限与审计证明，不存原始明文。
