# 脑域链医实施计划

## 状态说明
- `已完成`：代码、脚本或文档已落地，并完成过本地验证。
- `进行中`：已有骨架、样板链路或局部联调结果，但未完成全部目标。
- `未开始`：仍停留在规划阶段。

## 1. 目标与当前状态
- 目标是在 8 周内，从当前空仓库状态落地一个可运行、可测试、可演示的脑电数据安全共享平台，并完成 `3D 大脑热力图` 前端专项能力。
- 交付范围覆盖：智能合约、Java 业务后端、Python EEG/联邦学习服务、Vue 前端、基础设施编排、测试体系、演示脚本与交付文档。
- `已更新` 当前仓库已不再是空仓库，已经具备工程骨架和多条样板链路：
  - `已完成` monorepo 基线、环境文档、共享 OpenAPI、前后端/Python/合约最小可运行工程。
  - `已完成` 上传样板链路：`EDF/GDF/BDF 上传 -> 链下存储 -> 链上存证 -> 详情页查看`。
  - `已完成` 链路抽象：`ChainGateway` 与 `StorageGateway`。
  - `已完成` 本地 `FISCO BCOS 3.x` 联调，已验证真实写链。
  - `已完成` `MinIO` 存储网关，已验证真实对象上传。
  - `进行中` `brain-activity` 详情页可视化、时间轴与频段切换。
  - `未开始` DID/VC、RBAC、审计中心、FATE、销毁闭环、Three.js 真 3D 热力图。

## 2. 实施策略
- 架构上采用 `monorepo + 模块化单体 + 双 Python 服务 + 合约层`。在 8 周窗口内，不建议从零拆成多套微服务，以降低联调成本。
- Java 侧承接统一业务编排、认证鉴权、链/IPFS 调度、审计汇总与外部网关能力。
- Python 侧拆为两个独立服务：`eeg-service` 负责 EEG 解析/预处理/脑区活跃度计算，`federated-service` 负责 FATE 编排与训练归档。
- 前端维持单一 Vue 3 门户，按角色切换视图，不拆分多套站点。
- 首版坚持“链上只存证明、不存原始明文；前端只读聚合结果、不读原始 EEG”的边界。

## 3. 建议仓库结构
```text
BrainWeb3/
  apps/
    backend/                 # Spring Boot，业务编排与统一 API
    frontend/                # Vue 3 + Vite
  services/
    eeg-service/             # Flask + MNE，上传解析/预处理/脑区活跃度
    federated-service/       # Flask，FATE 编排与任务状态回传
  contracts/
    solidity/                # DataNotary / AccessControl / ContributionLedger / DestroyManager / DidRegistry
    deploy/                  # 部署、初始化、测试账户灌库脚本
  infra/
    docker/                  # MySQL / Redis / IPFS / FISCO BCOS / monitoring
    compose/                 # docker-compose.*.yml
  packages/
    shared-api/              # OpenAPI / DTO / 错误码 / 事件定义
    shared-config/           # 环境变量模板、lint、CI 复用配置
  scripts/
    bootstrap/               # 一键初始化、本地启动、演示脚本
  docs/
    architecture/
    api/
    security/
    testing/
```

## 6. 关键依赖与串行关系
- `合约完成` 是链适配层、审计中心、授权链路的前置条件。
- `DID/VC + JWT + RBAC` 是访问申请、训练启动、监管查询的前置条件。
- `EEG 预处理与脑区映射` 是 `brain-activity` 接口和 3D 热力图联调的前置条件。
- `数据授权校验` 是联邦训练编排上线的硬门槛，必须先于训练任务开放。
- `链上事件索引` 是审计中心与监管看板可用的前置条件。

## 10. 主要风险与应对
- `FISCO BCOS + FATE` 联调复杂度高：先分别做最小闭环，再进行平台级集成。
- `DID/VC` 实现范围容易膨胀：首版只做注册、签发、校验、吊销四项必需能力。
- `3D 大脑模型` 资源不可控：预留简化分区模型方案，不阻塞交互与数据联调。
- `跨语言链路` 排错成本高：统一请求追踪 ID、审计事件 ID、训练任务 ID。
- `空仓库起步` 容易拖慢第 1 周：必须把工程基线和样板链路作为第一优先级。
