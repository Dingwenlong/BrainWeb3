# 环境部署手册

## 1. 本地依赖
- Node.js `24+`
- npm `11+`
- Java `17+`
- Maven `3.9+`
- Python `3.10+`
- Docker `29+`

## 2. 初始化
1. 复制模板环境变量文件。
   `Copy-Item .env.example .env`
2. 执行环境检查脚本。
   `.\scripts\bootstrap\check-env.ps1`
3. 启动本地基础设施。
   `.\scripts\bootstrap\dev-up.ps1`

## 3. 模块启动
### 前端
```powershell
npm install
npm run dev:frontend
```

### 后端
```powershell
mvn -pl apps/backend spring-boot:run
```

### EEG 服务
```powershell
python -m pip install -r .\services\eeg-service\requirements.txt
python .\services\eeg-service\app.py
```

### 联邦服务
```powershell
python -m pip install -r .\services\federated-service\requirements.txt
python .\services\federated-service\app.py
```

### 本地 FISCO 联调链
```powershell
.\scripts\bootstrap\fisco-dev-up.ps1
```

Windows + Git Bash 下默认会使用 `C:\Users\jimmy\.brainweb3-fisco-dev` 作为链工作目录，并以 `1` 节点明文 RPC 模式联调；如需覆盖，可先设置：

```powershell
$env:BRAINWEB3_FISCO_ROOT='C:\your-shared-path\.brainweb3-fisco-dev'
```

生成完成后，可按脚本输出设置：
```powershell
$env:CHAIN_ENABLED='true'
$env:CHAIN_CONFIG_PATH='C:\Users\jimmy\.brainweb3-fisco-dev\sdk\config.toml'
```

详细说明见 [FISCO 本地联调手册.md](/D:/Devs/BrainWeb3/docs/FISCO%20%E6%9C%AC%E5%9C%B0%E8%81%94%E8%B0%83%E6%89%8B%E5%86%8C.md)。

### MinIO 链下对象存储
开发 compose 已包含 `MinIO`，默认端口：

- S3 API：`9000`
- Console：`9001`

如需切到 MinIO 存储网关，可在启动后端前设置：

```powershell
$env:STORAGE_PROVIDER='minio'
$env:MINIO_ENDPOINT='http://localhost:9000'
$env:MINIO_ACCESS_KEY='brainweb3'
$env:MINIO_SECRET_KEY='brainweb3secret'
$env:MINIO_BUCKET='brainweb3-datasets'
```

后端上传链路会在首次写入时自动创建 bucket，并把链下引用写成 `minio://bucket/object-key`。
