# 当前项目技术栈报告

## 1. 报告范围
- 本报告基于当前仓库 `BrainWeb3` 已实际落地的代码、配置和脚手架编写。
- 报告强调“当前真实状态”，不把设计文档中的远期目标直接写成已完成能力。
- 表格中的“技术选型”优先采用仓库中已声明的依赖版本、配置版本或镜像版本。

## 2. 项目当前总体判断
- 当前项目已经完成第 1 阶段工程骨架，并已打通一条可演示的样板业务链路：
  `数据集上传 -> 数据集列表/详情 -> 脑区活跃度查看`。
- 当前项目属于 `全栈 monorepo` 形态，已包含：
  - Vue 3 前端门户
  - Spring Boot 后端 API
  - 两个 Flask Python 服务
  - Solidity/Hardhat 合约工程
  - Docker Compose 本地基础设施
  - OpenAPI、CI、测试与启动脚本
- 当前尚未真正接入：
  - FISCO BCOS 3.x 联盟链节点、证书与真实写链接口
  - FATE 联邦学习运行环境
  - DID/VC 真实签发校验
  - IPFS/链上真实存证编排
  - EEG 真实预处理与 band power 计算链路

## 4. 按模块拆解的技术栈说明

### 4.1 前端层
| 层级 | 技术选型 | 说明 |
|---|---|---|
| 应用框架 | Vue 3 | 当前页面由 `App.vue`、`OverviewPage.vue`、`DatasetDetailPage.vue` 组成。 |
| 路由 | Vue Router | 当前已实现 `/` 和 `/datasets/:datasetId` 两个核心路由。 |
| 类型系统 | TypeScript | 当前前端 API 类型集中在 `src/types/api.ts`。 |
| 构建工具 | Vite | 当前本地开发和打包均由 Vite 驱动。 |
| API 调用 | Fetch | 当前通过 `src/api/client.ts` 封装 `GET/POST multipart` 请求。 |
| 样式实现 | 原生 CSS | 当前不依赖 Element Plus、Ant Design Vue 等 UI 库。 |
| 视觉方向 | 自定义“医疗科幻神经舱” | 当前界面风格是深色、青蓝高亮、橙色强调，不是默认脚手架页面。 |

### 4.2 后端层
| 层级 | 技术选型 | 说明 |
|---|---|---|
| 主框架 | Spring Boot 3.3.5 | 当前后端是统一 REST API 服务。 |
| Web 协议 | Spring MVC | 当前已实现系统状态、数据集列表、详情、上传、脑区活跃度代理。 |
| 配置管理 | `application.yml` + `@ConfigurationProperties` | 当前 EEG 服务地址和 FISCO 链路入口都通过配置属性管理。 |
| 服务调用 | RestTemplate | 当前后端调用 EEG 服务获取 `brain-activity` 数据。 |
| 链路方向 | FISCO BCOS 3.x Java SDK | 当前已新增 `brainweb3.chain.*` 配置和示例 TOML，但尚未真实连链。 |
| 领域组织 | `api / domain / service` | 当前 `dataset` 模块按接口、领域对象、服务拆分。 |
| 构建 | Maven | 当前命令为 `mvn -pl apps/backend test`、`spring-boot:run`。 |

### 4.3 Python 服务层
| 层级 | 技术选型 | 说明 |
|---|---|---|
| Web 框架 | Flask | 两个 Python 服务都使用 Flask 暴露 HTTP 接口。 |
| EEG 服务 | Flask Mock EEG API | 当前提供 `/health` 与 `/api/v1/datasets/{id}/brain-activity`。 |
| 联邦服务 | Flask Mock Training API | 当前提供 `/health` 与 `/api/v1/trainings/preview`。 |
| 测试 | pytest | 当前两个 Python 服务都可独立跑测试。 |

## 7. 结论
- 如果按“当前仓库已完成的真实技术栈”来定义，本项目已经是一个 `Vue 3 + Spring Boot + Flask + FISCO 3.x(Java SDK 方向) + Solidity/Hardhat(本地辅助) + Docker Compose` 的全栈 monorepo。
- 如果按“最终目标技术栈”来定义，本项目还会继续向 `FISCO BCOS + DID/VC + MNE + FATE + IPFS真实集成 + Three.js` 演进。
- 因此，当前最准确的描述应当是：
  `已完成工程骨架与样板链路的全栈系统，核心落地栈为 Vue 3 / Spring Boot / Flask / FISCO 3.x(Java SDK 方向) / Hardhat(本地辅助) / Docker Compose；真实节点、联邦学习和真实 EEG 处理仍处于下一阶段。`
