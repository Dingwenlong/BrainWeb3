## 2. 验收前准备

### 2.1 回归基线
- `npm run verify:ci`
- `npm run test:backend`
- `npm run test:python-services`

### 2.2 模拟正式环境预检
- `npm run check:prod-env -- -EnvFile .\.env.production.example -AllowExampleValues`

### 2.3 模拟验收报告样例
- `npm run report:acceptance -- -EnvFile .\.env.production.example -AllowExampleValues -SkipSmoke`

## 3. 核心验收路径

### 路径 A：研究员申请与归属方审批
账号：
- `researcher-01`
- `owner-01`

步骤：
1. 研究员登录
2. 打开数据集详情页
3. 提交访问申请
4. 归属方登录
5. 在审批台处理申请
6. 回到研究员视角确认门禁变化

验收点：
- 访问申请可创建
- 审批状态可变化
- 审计中心可回看动作
- 已批准后读取门禁生效

### 路径 B：研究员训练与模型治理主线
账号：
- `researcher-01`
- `admin-01` 或 `owner-01`

步骤：
1. 研究员登录
2. 从数据详情或训练页发起训练
3. 查看训练状态变化
4. 打开模型库
5. 查看模型治理状态、审计主线和链主线

验收点：
- 训练任务能创建并刷新状态
- 模型登记自动出现
- 模型治理总览可查看
- 审计中心和链轨迹可回看对应模型事件

### 路径 C：管理员账户与身份治理
账号：
- `admin-01`

步骤：
1. 管理员登录
2. 打开账户页
3. 查看账户目录
4. 打开身份中心
5. 查看账户 VC / 机构 VC / 历史时间线

验收点：
- 账户治理入口可用
- DID / VC 信息可展示
- 凭证状态与历史时间线可回看

### 路径 D：销毁闭环与凭证展示
账号：
- `owner-01` 或 `admin-01`

步骤：
1. 打开销毁台
2. 创建销毁申请
3. 审批或执行销毁
4. 查看物理清理状态
5. 查看清理凭证与链记录

验收点：
- 销毁申请流程可走通
- 数据状态会切换
- 审计和链轨迹能回看对应动作

### 路径 E：审计中心与链轨迹回放
账号：
- `admin-01`

步骤：
1. 打开审计中心
2. 按动作或数据集筛选
3. 打开链轨迹页
4. 查看链策略、链记录和跨页联动

验收点：
- 审计可按条件查询
- 链记录可按条件查询
- 从审计到链轨迹、从链轨迹到训练/审批/模型页的联动可用

## 4. 推荐最小验收组合

如果时间很紧，建议只跑这 3 条：

1. 路径 A：申请与审批
2. 路径 B：训练与模型治理
3. 路径 E：审计与链轨迹回放

这三条已经足够说明平台的核心闭环。

## 5. 结果留档建议

每次验收建议至少保留：

- 本轮使用的环境文件说明
- 本轮使用的账号
- 一份 `report:acceptance` 输出
- 关键页面截图
- 若有失败项，附一句原因说明

建议在结果留档后，再执行：

```powershell
npm run package:delivery -- -EnvironmentLabel simulated-prod
```

这样可以把验收报告、演示脚本、账号清单、验收路径、环境模板和截图目录统一收成一个交付包。
