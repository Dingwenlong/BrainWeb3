# P5 正式环境联调清单

更新日期：`2026-04-22`

## 1. 适用范围
这份清单面向 `P5-2` 阶段，用来把当前仓库从“开发机可跑”推进到“第三方按步骤可联调”的状态。

目标不是直接给出生产级部署方案，而是先固定：

1. 哪些参数必须由环境提供
2. 哪些组件必须先准备好
3. 后端、前端、链和链下存储之间如何对接

## 2. 推荐联调拓扑
- 前端：`apps/frontend` 静态资源，反向代理到浏览器入口
- 后端：`apps/backend`，对接 MySQL、MinIO、EEG 服务、FISCO BCOS
- EEG 服务：`services/eeg-service`
- 联邦服务：当前仍为 mock 编排，不作为正式环境阻塞项
- 基础设施：`MySQL + Redis + MinIO + IPFS`
- 链环境：`FISCO BCOS 3.x` 节点与 SDK 证书目录

## 3. 参数映射
### 应用层
- `APP_ENV`
  建议值：`production` 或 `staging`
- `APP_STAGE`
  建议值：`production`、`staging`
- `APP_PORT`
  后端监听端口，默认 `8080`

### 数据库
- `SPRING_DATASOURCE_URL`
  正式环境建议显式提供，不再依赖 `MYSQL_*` 自动拼接
- `SPRING_DATASOURCE_USERNAME`
- `SPRING_DATASOURCE_PASSWORD`

### 认证与身份
- `AUTH_JWT_SECRET`
  必须替换，不能继续使用仓库默认值
- `AUTH_DEMO_PASSWORD`
  正式环境建议禁用演示密码语义
- `AUTH_ALLOW_DEMO_BOOTSTRAP=false`
- `AUTH_ALLOW_DEMO_PASSWORD_LOGIN=false`
- `IDENTITY_CREDENTIAL_SECRET`
  必须替换
- `IDENTITY_ISSUER_DID`
  建议区分 `staging / production`

### EEG 服务
- `EEG_SERVICE_BASE_URL`
  指向真实 EEG 服务地址，例如 `http://eeg-service.internal:8101`

### 链环境
- `CHAIN_ENABLED=true`
- `CHAIN_CONFIG_PATH`
  指向 SDK `config.toml`
- `CHAIN_CONTRACT_ADDRESS`
  指向已部署 `DataNotary`
- `CHAIN_BUSINESS_CONTRACT_ADDRESS`
  指向已部署业务事件合约
- `CHAIN_AUTO_DEPLOY=false`
- `CHAIN_BUSINESS_CONTRACT_AUTO_DEPLOY=false`

### 对象存储
- `STORAGE_PROVIDER=minio`
- `MINIO_ENDPOINT`
- `MINIO_ACCESS_KEY`
- `MINIO_SECRET_KEY`
- `MINIO_BUCKET`
- `MINIO_AUTO_CREATE_BUCKET=false`

## 4. 推荐联调顺序
在真正启动前，建议先把 `.env.production` 放到仓库根目录，然后执行：

```powershell
npm run check:prod-env
```

若当前环境还没有联调账号，建议先按 [P5 联调账号准备说明.md](/D:/Devs/BrainWeb3/docs/P5%20联调账号准备说明.md) 生成并导入账号种子 SQL。

推荐命令顺序：

```powershell
npm run seed:accounts -- -Profile standard-demo-roles -DefaultPassword <联调密码>
npm run import:account-seed -- -EnvFile .\.env.production -SqlPath .\artifacts\account-seeds\<生成的 SQL 文件>
```

如果当前机器也能直连正式环境 MySQL、EEG 服务和 MinIO，可以进一步执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\bootstrap\check-prod-env.ps1 -EnvFile .\.env.production -CheckReachability
```

预检通过后，建议再用一个联调账号执行后端 smoke：

```powershell
npm run smoke:backend -- -BaseUrl http://backend.internal:8080 -ActorId <联调账号> -Password <联调密码>
```

这一步会按顺序检查：
- `/api/v1/system/status`
- `/api/v1/auth/login`
- `/api/v1/auth/session`
- `/api/v1/accounts/me`
- `/api/v1/identity/me`
- `/api/v1/datasets`
- 若账号为 `admin / owner / approver`，还会继续检查 `audits / training-jobs / model-records / chain-records`

如果需要把本轮联调结果直接沉成材料，可进一步执行：

```powershell
npm run report:acceptance -- -EnvFile .\.env.production -BaseUrl http://backend.internal:8080 -ActorId <联调账号> -Password <联调密码>
```

该命令会在 `artifacts/acceptance/` 下输出：
- preflight JSON
- smoke JSON
- Markdown 验收记录

如果需要把本轮联调结果和演示/验收材料一起整理成可交付目录，可继续执行：

```powershell
npm run package:delivery -- -EnvironmentLabel production
```

该命令会在 `artifacts/delivery/` 下生成一个带时间戳的交付目录，便于提交老师、甲方或验收方。

### 第一步：基础设施
先准备：
- MySQL
- Redis
- MinIO
- IPFS
- FISCO BCOS 3.x

如果只是预演环境，可继续使用 [环境部署手册.md](/D:/Devs/BrainWeb3/docs/%E7%8E%AF%E5%A2%83%E9%83%A8%E7%BD%B2%E6%89%8B%E5%86%8C.md) 里的本地 compose 作为基础设施替身。

### 第二步：链配置
按 [FISCO 本地联调手册.md](/D:/Devs/BrainWeb3/docs/FISCO%20%E6%9C%AC%E5%9C%B0%E8%81%94%E8%B0%83%E6%89%8B%E5%86%8C.md) 准备：
- `config.toml`
- SDK 证书目录
- 已部署合约地址

正式环境联调时，建议显式设置：

```powershell
$env:CHAIN_ENABLED='true'
$env:CHAIN_AUTO_DEPLOY='false'
$env:CHAIN_BUSINESS_CONTRACT_AUTO_DEPLOY='false'
$env:CHAIN_CONFIG_PATH='D:\deploy\fisco\config.toml'
$env:CHAIN_CONTRACT_ADDRESS='replace-with-data-contract'
$env:CHAIN_BUSINESS_CONTRACT_ADDRESS='replace-with-business-contract'
```

### 第三步：对象存储
建议把正式环境默认存储切到 `MinIO`：

```powershell
$env:STORAGE_PROVIDER='minio'
$env:MINIO_ENDPOINT='https://minio.internal:9000'
$env:MINIO_ACCESS_KEY='replace-me'
$env:MINIO_SECRET_KEY='replace-me'
$env:MINIO_BUCKET='brainweb3-datasets'
$env:MINIO_AUTO_CREATE_BUCKET='false'
```

### 第四步：后端
启动前至少确认以下变量已生效：

```powershell
$env:APP_ENV='production'
$env:APP_STAGE='production'
$env:SPRING_DATASOURCE_URL='jdbc:mysql://mysql.internal:3306/brainweb3?createDatabaseIfNotExist=true&useSSL=true&allowPublicKeyRetrieval=false&serverTimezone=UTC'
$env:SPRING_DATASOURCE_USERNAME='brainweb3_app'
$env:SPRING_DATASOURCE_PASSWORD='replace-me'
$env:AUTH_JWT_SECRET='replace-with-prod-secret'
$env:AUTH_DEMO_PASSWORD='replace-with-non-demo-password'
$env:AUTH_ALLOW_DEMO_BOOTSTRAP='false'
$env:AUTH_ALLOW_DEMO_PASSWORD_LOGIN='false'
$env:IDENTITY_CREDENTIAL_SECRET='replace-with-prod-identity-secret'
$env:EEG_SERVICE_BASE_URL='http://eeg-service.internal:8101'
```

然后执行：

```powershell
mvn -pl apps/backend spring-boot:run
```

### 第五步：前端
前端继续使用现有构建产物或静态站点部署方式。联调时至少确认：
- 前端可访问后端 API
- 登录页可完成真实登录
- 总览页、训练页、审计页、链轨迹页都能加载

## 5. 联调通过标准
至少完成以下检查：

1. 后端成功连接 MySQL 并执行 Flyway
2. 数据上传可以写入 MinIO 或正式链下存储
3. `CHAIN_ENABLED=true` 时，至少一条链写入能拿到真实 `tx hash`
4. 登录、审计、训练、销毁、模型治理主链路都能在页面中回看
5. `npm run test:backend`、`npm run verify:ci`、`npm run test:python-services` 作为回归基线保持可通过
6. `npm run check:prod-env` 通过，且不再依赖人工逐条核对默认 secret / demo 开关 / 关键地址
7. `npm run smoke:backend -- -BaseUrl ... -ActorId ... -Password ...` 通过，能证明正式环境最小 API 主链路可访问

## 6. 当前边界
- 联邦服务仍是 mock 编排层，`P5-2` 不以真实 FATE 环境为阻塞
- 这份清单解决的是“联调基线”，不是完整生产部署、监控、备份和容灾方案
- 演示密码、默认 secret、H2/Flyway 测试 warning 等安全与生产化问题，会在后续 `P5-3` 单独收口
