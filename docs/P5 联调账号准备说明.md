# P5 联调账号准备说明

更新日期：`2026-04-22`

## 1. 适用范围

这份说明用于 `staging / production` 或准正式联调环境的账号准备。

目标是解决两个问题：

1. demo 账号在正式口径下不能再自动灌库
2. smoke、验收和演示仍然需要一组稳定可复用的联调账号

## 2. 为什么不能继续用 demo bootstrap

正式环境变量模板已经要求：

- `AUTH_ALLOW_DEMO_BOOTSTRAP=false`
- `AUTH_ALLOW_DEMO_PASSWORD_LOGIN=false`

也就是说，进入正式口径后：

- 后端不会再自动创建 `researcher-01 / owner-01 / approver-01 / admin-01`
- 也不应该继续使用 `brainweb3-demo` 这类默认密码

## 3. 推荐账号结构

推荐沿用当前演示账号结构，但替换成单独分发的联调密码：

- `researcher-01`
- `owner-01`
- `approver-01`
- `admin-01`

好处是：

- 前端演示路径、验收脚本和说明文档都不用改
- smoke、演示和验收说法一致
- 只替换密码，不会影响角色理解

## 4. 两种准备方式

### 方式 A：先只准备一个初始管理员

适合场景：

- 你们希望后续账号都通过后台页面或 `/api/v1/accounts` 创建
- 你们希望正式环境里先只放一个管理员

命令：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\bootstrap\build-account-seed-sql.ps1 `
  -Profile admin-only `
  -AdminActorId admin-01 `
  -AdminDisplayName "平台管理员" `
  -AdminOrganization "Huaxi Medical Union" `
  -AdminPassword "ReplaceWithStrongAdminPassword#2026"
```

### 方式 B：一次准备标准联调 4 账号

适合场景：

- 你想直接跑 smoke、演示和验收
- 你想保留当前仓库里的账号命名

命令：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\bootstrap\build-account-seed-sql.ps1 `
  -Profile standard-demo-roles `
  -DefaultPassword "ReplaceWithStrongSharedPassword#2026"
```

如果希望不同角色使用不同密码，可以显式传：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\bootstrap\build-account-seed-sql.ps1 `
  -Profile standard-demo-roles `
  -AdminPassword "AdminPassword#2026" `
  -ResearcherPassword "ResearcherPassword#2026" `
  -OwnerPassword "OwnerPassword#2026" `
  -ApproverPassword "ApproverPassword#2026"
```

## 5. 生成结果是什么

脚本会输出一个可直接给 MySQL 执行的 SQL 文件，默认在：

`artifacts/account-seeds/`

特点：

- 密码会按后端同口径生成 `BCrypt` 哈希
- 同一个 `actorId` 已存在时，会执行更新而不是报错
- 重跑脚本会重置这些账号的密码和更新时间

## 6. 推荐执行顺序

1. 生成 SQL 文件
2. 在目标 MySQL 执行 SQL
3. 启动后端
4. 用 `admin-01` 或你指定的管理员账号登录
5. 执行 `npm run smoke:backend -- -BaseUrl ... -ActorId ... -Password ...`
6. 再执行 `npm run report:acceptance`

如果本机已安装 `mysql` 客户端，也可以直接走仓库脚本导入：

```powershell
npm run import:account-seed -- -EnvFile .\.env.production -SqlPath .\artifacts\account-seeds\<生成的 SQL 文件>
```

若当前只想先确认解析出来的目标库和命令是否正确，可先执行：

```powershell
npm run import:account-seed -- -EnvFile .\.env.production -SqlPath .\artifacts\account-seeds\<生成的 SQL 文件> -PrintOnly
```

## 7. 注意事项

- 不要把生成后的 SQL 文件提交到仓库，因为里面包含密码哈希
- 如果你使用 `-DefaultPassword`，请确认这是仅用于联调的受控密码
- 若环境已经有正式账户体系，建议只准备一个最小联调账号集合，不要混入真实生产人员账号
- `admin-only` 更适合长期环境，`standard-demo-roles` 更适合阶段验收和演示
