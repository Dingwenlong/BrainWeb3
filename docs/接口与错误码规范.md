# 接口与错误码规范

## 1. 基础约定
- 基础路径：`/api/v1`
- 响应格式建议统一为：

```json
{
  "success": true,
  "traceId": "req-20260420-0001",
  "data": {}
}
```

错误响应：

```json
{
  "success": false,
  "traceId": "req-20260420-0001",
  "error": {
    "code": "DATASET_NOT_FOUND",
    "message": "Dataset 42 does not exist."
  }
}
```

## 2. 错误码分段
- `AUTH_*`：认证与会话错误
- `DID_*`：DID/VC 注册、签发、校验错误
- `DATASET_*`：上传、校验、存证、查询错误
- `ACCESS_*`：访问申请、审批、撤销错误
- `TRAINING_*`：联邦训练编排与状态错误
- `AUDIT_*`：审计检索与追踪错误
- `DESTROY_*`：销毁流程错误
- `INFRA_*`：MySQL、Redis、IPFS、链节点等依赖异常

## 3. 首批保留错误码
- `AUTH_INVALID_TOKEN`
- `AUTH_PERMISSION_DENIED`
- `DID_CREDENTIAL_EXPIRED`
- `DATASET_INVALID_FILE_FORMAT`
- `DATASET_HASH_MISMATCH`
- `DATASET_STORAGE_UPLOAD_FAILED`
- `ACCESS_REQUEST_NOT_APPROVED`
- `TRAINING_DATASET_NOT_AUTHORIZED`
- `AUDIT_TRACE_NOT_FOUND`
- `DESTROY_PROOF_NOT_FOUND`

## 4. 跨服务约定
- 所有服务日志都必须带 `traceId`。
- Java 后端作为网关时负责生成或透传 `traceId`。
- Python 服务返回错误时保留稳定错误码，不直接暴露内部堆栈。
