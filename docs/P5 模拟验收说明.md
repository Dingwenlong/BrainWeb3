# P5 模拟验收说明

更新日期：`2026-04-22`

## 1. 当前口径

由于当前项目尚无可用的正式环境，`P5` 阶段的完成定义调整为：

`在模拟正式环境配置下，完成可验证、可留档、可重复执行的验收。`

这意味着当前 `P5` 不是要求：

- 真实生产地址全部跑通
- 真实正式链、正式对象存储、正式数据库都已接入

而是要求：

- 用正式环境口径配置完成预检
- 用模拟正式环境或准正式环境完成 smoke
- 能产出结构化验收报告
- 能明确说明“当前已完成什么、缺什么、接入正式环境还要补什么”

## 2. 模拟验收包含什么

### 2.1 配置层
- 使用 `.env.production.example` 或按其格式整理出的模拟正式配置
- 关闭 demo bootstrap 与 demo password 登录
- 替换默认 secret
- 保持 `production / staging` 口径下的启动门禁有效

### 2.2 验证层
- `npm run verify:ci`
- `npm run test:backend`
- `npm run test:python-services`
- `npm run check:prod-env`
- `npm run smoke:backend`
- `npm run report:acceptance`

### 2.3 材料层
- 演示账号清单
- 验收路径清单
- 模拟验收报告
- 已知边界与迁移清单

## 3. 当前通过标准

满足以下条件即可认为“模拟正式环境验收完成”：

1. 本地和 CI 回归命令均固定
2. 模拟正式环境配置可以通过 `check:prod-env`
3. 至少一套联调账号可以通过 `smoke:backend`
4. 能生成 Markdown + JSON 形式的验收报告
5. 项目边界与未完成项已在文档中显式说明

## 4. 这不等于什么

当前口径不等于：

- 真实生产联调完成
- 真实 FATE 环境完成
- 真实 VC 正式签名体系完成
- 真实链上全部业务合约与索引能力完成
- 完整运维、安全、监控体系完成

这些能力仍属于后续“接入正式环境”或“完整产品化”阶段。

## 5. 当前建议使用方式

如果近期目标是汇报、验收、答辩、阶段检查，建议统一使用下面这组说法：

- 当前项目已完成 `P5 模拟正式环境验收`
- 当前尚未进入真实正式环境联调
- 项目已经具备接入正式环境前的大部分代码、脚本与材料基础

## 6. 和后续正式环境联调的关系

当后续获得真实正式环境后，不需要重做整套 `P5`，只需要补一个短收口：

1. 用真实 `.env.production` 重跑预检
2. 用真实联调账号重跑 smoke
3. 用真实基础设施产出最终版验收报告

也就是说，当前这套模拟验收材料可以直接作为正式环境联调前的基线。
