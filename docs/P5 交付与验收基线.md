# P5 交付与验收基线

更新日期：`2026-04-22`

## 1. 阶段目标
`P5` 不再以“功能有没有”作为主目标，而是以“现有闭环能否持续验证、模拟正式环境验收和交付”作为主目标。

本阶段优先收口四条线：

1. 统一 CI 与本地回归基线
2. 模拟正式环境联调与配置清单
3. 安全与运维最小收口
4. 演示与验收材料整理

## 2. 当前默认回归口径
### 前端与合约
在仓库根目录执行：

```powershell
npm run verify:ci
```

覆盖内容：
- `apps/frontend` 构建
- `apps/frontend` 单元测试
- `contracts/solidity` 合约测试

### 后端
在仓库根目录执行：

```powershell
npm run test:backend
```

覆盖内容：
- `apps/backend` 全量 Spring Boot 测试

### Python 服务
在仓库根目录执行：

```powershell
npm run test:python-services
```

覆盖内容：
- `services/eeg-service/tests`
- `services/federated-service/tests`

## 3. GitHub Actions 基线
当前 `.github/workflows/ci.yml` 已按以下维度执行：

1. 后端 Maven 测试
2. 前端构建 + 前端测试 + 合约测试
3. EEG 服务与联邦服务的 Python 测试

这意味着一个变更在进入 `main/master` 之前，至少会经过一次全栈基础回归，而不是只做前端 build 或单个模块测试。

## 4. P5 推荐执行顺序
### P5-1：验证基线
- 固定本地验证命令
- 固定 GitHub Actions 回归口径

### P5-2：模拟正式环境联调

- 明确后续真实 `FISCO BCOS / MinIO / MySQL / Redis` 的接入参数

### P5-3：安全与运维收口
- 默认密钥、演示密码、开发模式开关清点
- 鉴权、链写入、销毁、模型治理等关键接口做最小权限复核
- 明确日志、审计、错误提示里的敏感信息边界

当前已落地的第一批安全门禁：
- `production / staging` 等正式环境口径下，默认 JWT secret 会阻止后端启动
- 默认 identity credential secret 会阻止后端启动
- demo 账户自动引导和 demo 密码登录需要显式关闭，否则会阻止后端启动

### P5-4：验收与演示材料
- 演示账号清单
- 演示脚本
- 验收路径清单
- 功能截图与风险说明
- 已知限制与迁移清单

当前已补齐统一整理出口：
- `npm run package:delivery`
- 输出目录：`artifacts/delivery/`
- 输出内容：验收报告、联调清单、演示脚本、账号清单、验收路径、边界说明、环境模板、证据目录

## 5. 当前退出标准
满足以下条件后，可以认为 `P5` 基本完成：

1. 本地与 CI 的统一回归入口已经固定
2. 模拟正式环境联调文档可支撑第三方按步骤执行验收
3. 核心安全边界与默认配置完成一次显式复核
4. 验收人员可以按脚本完成一轮主链路演示

## 6. 当前已知非阻塞项
- 当前没有显式挂起的基础回归 warning；后续风险主要会集中在模拟验收与真实正式环境之间的配置差异
