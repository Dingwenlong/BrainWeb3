## 2. 演示前固定动作

### 2.1 验证命令
- `npm run verify:ci`
- `npm run test:backend`
- `npm run test:python-services`
- `npm run check:prod-env -- -EnvFile .\.env.production`
- `npm run smoke:backend -- -BaseUrl <后端地址> -ActorId <联调账号> -Password <联调密码>`

### 2.2 本轮材料
- 一份最新 `report:acceptance` 产物
- 一份本轮环境说明
- 一组演示账号
- 关键页面截图或录屏目录

### 2.3 建议默认账号顺序
- `researcher-01`
- `owner-01`
- `admin-01`

## 3. 推荐 12 分钟演示顺序

### 第 1 步：开场与状态说明
话术：

`当前系统已经完成脑电数据上传、授权审批、训练、审计、链存证和身份治理的最小真实闭环。本轮演示按照正式环境验收口径进行，先看业务主线，再看审计与治理回放。`

屏幕建议：
- 总览页
- 或验收报告首页

验收点：
- 能说清本轮环境口径
- 能说清本轮账号和范围

### 第 2 步：研究员登录与数据查看
账号：
- `researcher-01`

操作：
1. 登录
2. 打开总览页
3. 进入数据详情页
4. 展示 3D 热力图、频段切换和指标面板

验收点：
- 登录可用
- 数据详情可打开
- 脑区分析和时间轴可交互

兜底话术：

`如果当前样本数据较少，这里主要验证的是工作台、数据门禁和分析链路已经打通，而不是追求大样本展示。`

### 第 3 步：研究员发起访问申请
账号：
- `researcher-01`

操作：
1. 在数据详情页或申请入口提交访问申请
2. 说明申请用途和最小必要范围

验收点：
- 申请能创建
- 状态会进入待审批
- 动作会进入审计主线

兜底话术：

`如果现场网络较慢，审批结果稍后会在审计中心和链轨迹里一起回看，不影响闭环判断。`

### 第 4 步：归属方审批
账号：
- `owner-01`

操作：
1. 切换账号
2. 打开审批台
3. 审批刚才的申请
4. 返回研究员视角确认授权变化

验收点：
- 审批台可查询记录
- 审批状态可变更
- 研究员门禁随审批结果更新

兜底话术：

`如果页面缓存尚未刷新，可以用审计中心或重新进入详情页确认最新状态，系统判定以服务端状态为准。`

### 第 5 步：研究员发起训练并查看模型记录
账号：
- `researcher-01`

操作：
1. 进入训练页或从数据页发起训练
2. 查看训练任务状态
3. 打开模型库确认记录出现

验收点：
- 训练任务能创建
- 状态能刷新
- 模型登记与治理摘要可查看

兜底话术：

`当前联邦学习仍是最小训练闭环，这一步重点展示任务编排、日志摘要和模型登记链路已经接上。`

### 第 6 步：管理员查看审计与链轨迹
账号：
- `admin-01`

操作：
1. 打开审计中心
2. 按动作或数据集筛选
3. 打开链轨迹页
4. 展示与申请、训练、治理相关的链记录

验收点：
- 审计可按条件查询
- 链记录可回看
- 审计与链页面能解释同一业务动作

兜底话术：

`当前链能力以最小真实业务链路为主，重点是证明链上留痕、链下回看和策略分层已经贯通。`

### 第 7 步：管理员查看身份治理
账号：
- `admin-01`

操作：
1. 打开账户页
2. 进入身份中心
3. 展示账户 DID、机构 VC、状态快照和历史时间线

验收点：
- 账户治理入口可用
- 身份信息和状态历史可展示

兜底话术：

`当前 DID / VC 已能支撑平台内治理，后续正式版会继续补签发、验签和吊销注册表。`

### 第 8 步：收尾说明
话术：

`本轮演示证明平台已具备可验收的最小真实闭环，当前剩余工作主要集中在正式环境接入、运维基线强化，以及把联邦学习、DID/VC 和链能力从最小版升级到正式版。`

## 5. 异常处理原则

- 页面短暂失败时，优先刷新当前页或重新查询，不现场改代码
- 如链服务不可用，明确说明当前业务页与审计页仍可作为链下验收基线
- 如正式环境尚未接入，明确说明本轮属于“模拟正式环境验收”而不是“正式生产联调完成”
- 所有临时降级说明，都应在本轮验收报告或交付包备注里留痕
