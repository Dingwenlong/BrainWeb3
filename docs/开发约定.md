# 开发约定

## 1. 仓库分层
- `apps`：用户直接访问的应用层
- `services`：领域计算或编排型服务
- `contracts`：智能合约与部署脚本
- `packages`：共享接口定义、规范与复用配置
- `docs`：设计、接口、安全、测试、部署文档

## 2. 命名约定
- REST API 统一以 `/api/v1` 作为前缀。
- 资源名使用复数名词，例如 `/datasets`、`/access-requests`、`/trainings`。
- 链上事件使用过去式，例如 `DataRegistered`、`AccessGranted`。
- 统一业务对象命名：`Organization`、`User`、`Dataset`、`AccessRequest`、`TrainingJob`、`AuditEvent`。

## 3. 提交与分支
- 新分支默认使用 `codex/` 前缀。
- 每次提交围绕单一主题，避免把基础设施、业务逻辑、UI 大改混在同一个提交中。
- 不提交真实密钥、凭证、患者原始数据、链节点私钥。

## 4. 代码风格
- Java 使用 `Java 17` 语法级别，保持包结构清晰，Controller 只负责协议层。
- Python 使用类型注解和 app factory 风格，避免在模块导入时执行重逻辑。
- Vue 使用组合式 API、`script setup` 和 TypeScript。
- 合约优先保持事件、结构体、错误类型语义明确。

## 5. 质量基线
- 新增接口必须同时更新 OpenAPI 或共享接口文档。
- 核心流程至少补单元测试或最小接口测试。
- 对外错误统一使用稳定错误码，不直接把底层异常文本原样透出。
