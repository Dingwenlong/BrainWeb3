# P5 已知边界与正式环境迁移清单

更新日期：`2026-04-22`

## 1. 当前已知边界

### 1.1 联邦学习
- 当前是最小训练闭环
- 当前不是完整真实 `FATE` 生产联邦环境
- 当前不覆盖完整贡献度与模型生产治理体系

### 1.2 DID / VC
- 当前已完成最小 DID / VC 治理闭环
- 当前未完成真实签名 / 验签 / 对外交换格式
- 当前未完成更正式的吊销注册表机制

### 1.3 链能力
- 当前已具备可用的链记录、链轨迹、策略与失败补偿
- 当前仍以“最小真实链闭环” 为主
- 当前未完成更完整的正式业务事件合约体系与索引化治理

### 1.4 销毁与模型治理
- 当前销毁闭环已具备逻辑销毁、物理清理最小补偿和凭证展示
- 当前模型治理闭环已具备登记、状态机、审计主线和链主线
- 两者仍偏“可验收最小版”，不是完整生产治理平台

### 1.5 安全与运维
- 当前已完成默认 secret、demo 登录、敏感响应、敏感文本脱敏等最小安全收口
- 当前未完成完整安全测试、监控、备份、容灾、TLS / mTLS、系统级限流

## 2. 接入正式环境前必须做的事

### 2.1 配置与密钥
- 准备真实 `.env.production`
- 替换数据库、JWT、identity、MinIO 相关 secret
- 关闭 demo bootstrap 与 demo password 登录

### 2.2 基础设施
- 准备真实 MySQL
- 准备真实 MinIO 或明确正式链下存储策略
- 准备真实 EEG 服务地址
- 准备真实链节点与真实合约地址

### 2.3 联调验证
- 跑 `npm run check:prod-env`
- 跑 `npm run smoke:backend`
- 跑 `npm run report:acceptance`
- 留存最终版验收报告

## 3. 接入正式环境后优先补的事

### 第一优先级
- 真实基础设施连通
- 真实链写入与真实 `tx hash`
- 正式账号与密码分发
- 正式环境验收报告

### 第二优先级
- 真实 FATE 环境
- 更完整 DID / VC 正式化能力
- 更完整链上业务合约与事件治理

### 第三优先级
- 安全测试
- 监控与告警
- 备份与容灾
- 更系统的运维手册

## 4. 当前建议对外表述

如果需要对老师、领导、甲方或评审说明项目状态，建议使用下面这种说法：

`当前系统已完成核心业务闭环和模拟正式环境验收，但尚未进入真实正式环境联调。`

不要表述成：

- 已完成生产部署
- 已完成真实正式环境联调
- 已完成完整产品化

## 5. 一句话总结

当前项目最大的剩余工作不是“补基础功能”，而是：

`把现有最小闭环接入真实正式环境，并把少数最小版能力升级为正式版。`
